# TickFlow市场数据查询SOP（2026版）

## 适用范围
适用于A股、港股、美股、国内期货等市场的行情、K线、财务、基本面数据查询场景，为量化分析、趋势判断提供可靠数据支撑。

## 数据使用优先级
1. 第一优先级：TickFlow Skill获取的官方数据，所有行情类分析优先使用该数据源
2. 第二优先级：Qveris公开API获取的数据，仅作为交叉验证使用
3. 交叉验证规则：相同指标两份数据源差异超过0.5%时，重新核查查询参数，仍有差异时以TickFlow数据为准

## 常用查询场景规范
| 数据类型 | 数据源 | 验证要求 |
| --- | --- | --- |
| 实时行情 | TickFlow | 无需额外验证 |
| 历史K线（分钟/日/周/月） | TickFlow | 和Qveris对比1-2个校验点，差异<0.5%即可使用 |
| 上市公司财务数据 | TickFlow | 和Qveris对比最新一期数据，差异<1%即可使用 |
| 宏观经济指标 | Qveris | 标注官方发布机构 |
| 商品期货行情 | TickFlow | 和Qveris对比主力合约价格，差异<0.3%即可使用 |

## 查询规范
1. 所有查询必须使用标准标的代码格式：`代码.市场后缀`（如600000.SH、AAPL.US、au2606.SHF）
2. 批量查询超过100个标的时，使用TickFlow的batch接口，避免重复请求
3. 查询结果优先使用`as_dataframe=True`返回pandas格式，方便后续分析
4. 敏感数据查询需留痕，保存查询时间、参数、结果哈希值备查

## 异常处理
1. TickFlow接口不可用时，临时使用Qveris数据，标注"临时数据源"，恢复后重新验证修正
2. 数据差异过大时，优先核查标的代码、查询周期、复权方式等参数是否正确
3. 无法判断数据准确性时，同时披露两份数据源结果并说明差异
