## 升级一：L2 风格因子从 3 因子升级为 Fama-French 5 因子

### 1. 我们现在怎么做的？

L2 有三个因子，靠人工估计：

```
动量 5.00 ← "价格在涨，趋势很强"
价值 1.50 ← "CAPE 36.48，太贵了"
成长 4.25 ← "AI capex $725B，增速快"
```

**问题：** 这三个分数是我根据新闻和数据"拍"的。如果换一个人、换一天，分数可能不一样。缺乏可重复的计算公式。

### 2. Fama-French 是什么？

Fama 和 French 两位经济学家在 1990 年代发现，股票回报不只是"大盘涨我就涨"，背后有几个系统性因素在驱动。他们把这些因素叫做**因子**。

**用餐厅类比理解 5 个因子：**

| 因子 | 学术名 | 通俗解释 | 餐厅类比 |
|---|---|---|---|
| **市场** | Market Beta | "大盘涨我也涨"的联动程度 | 整条美食街的客流 |
| **规模** | SMB (小减大) | 小公司长期跑赢大公司 | 小馆子比连锁店增长快 |
| **价值** | HML (高减低) | 便宜股票长期跑赢贵股票 | 被低估的苍蝇馆子 vs 网红店 |
| **盈利** | RMW (强减弱) | 高利润公司跑赢低利润公司 | 利润率高的小店 vs 亏钱烧流量的店 |
| **投资** | CMA (保守减激进) | 不乱烧钱的公司跑赢疯狂扩张的 | 稳扎稳打的店 vs 疯狂开分店的 |

### 3. 我们现在 3 因子 vs FF 5 因子对比

```
我们现在的 3 因子                FF 5 因子
┌──────────────┐          ┌──────────────────────┐
│ 动量         │    →     │ 新增：动量因子(UMD)  │
│ (感觉+MA+RSI)│          │ (12月收益率-1月收益率) │
├──────────────┤          ├──────────────────────┤
│ 价值         │    →     │ 市场β + 规模(SMB)    │
│ (只看CAPE)   │          │ + 价值(HML)           │
├──────────────┤          │ + 盈利(RMW)           │
│ 成长         │    →     │ + 投资(CMA)           │
│ (看AI capex) │          │                       │
└──────────────┘          └──────────────────────┘
  手工粗估                    可计算的公式
```

### 4. 具体的计算公式（用你的 ETF 举例）

以 QQQ 为例，FF 5 因子的计算过程：

**数据需求：** 每月收益率 + Fama-French 因子数据（可从 Kenneth French 官网免费下载）

**计算公式：**

```
QQQ超额收益 = α + β_market × 市场因子 + β_size × 规模因子
             + β_value × 价值因子 + β_profit × 盈利因子
             + β_invest × 投资因子
```

**怎么读到这些数字：**

| 系数 | 如果 QQQ 的系数是 | 含义 |
|---|---|---|
| β_market | 1.2 | QQQ 比大盘波动大 20%——上涨时多涨，下跌时多跌 |
| β_size | -0.3 | QQQ 偏大公司（负值=大公司）——符合预期 |
| β_value | -0.5 | QQQ 偏成长（负值=非价值）——符合预期 |
| β_profit | +0.4 | QQQ 成分股盈利质量好 |
| β_invest | -0.2 | QQQ 成分股投资偏保守（科技公司轻资产） |

**拿到这些系数后，L2 风格因子就升级为：**

```
动量因子 = 计算 4 只 ETF 的 12-1 月动量取均值
市场因子 = 4 只 ETF 的 β_market 加权均值
规模因子 = 4 只 ETF 的 β_size 加权均值
价值因子 = 4 只 ETF 的 β_value 加权均值
盈利因子 = 4 只 ETF 的 β_profit 加权均值
投资因子 = 4 只 ETF 的 β_invest 加权均值
```

然后根据每个因子得分所处的历史百分位给出 1-5 评分。

### 5. 升级后的效果

| 维度 | 升级前 | 升级后 |
|---|---|---|
| 可重复性 | 每次手动估计 | 公式计算，任何时间结果一致 |
| 精细度 | 3 个笼统维度 | 6 个可量化的风险敞口 |
| 归因能力 | 只知道"涨了" | 知道"因为什么因子涨了" |
| ETF 间比较 | 感觉 QQQ 比创业板强 | 量化说出差在哪个因子上 |
| 学术支撑 | 无 | 30 年学术验证，Nobel 奖级别 |

### 6. 实现路径

```
Step 1: 从 Kenneth French 官网下载 FF 5 因子月度数据
Step 2: 拿 4 只 ETF 的日线数据计算月度收益（TickFlow）
Step 3: 用 Python statsmodels 跑回归，得到每个 ETF 的 5 个 β
Step 4: 计算动量因子（12 个月收益 - 最近 1 个月收益）
Step 5: 将所有 β 映射到 1-5 评分表
Step 6: 写入 L2 风格因子评分卡
```

**预计工作量：** 编写一个 Python 脚本（约 200 行），运行时间 < 5 秒。

## 升级二：L3 情绪面加入 VaR/CVaR 量化风险

### 1. 我们现在怎么做的？

L3 情绪温度用了 VIX、Put/Call、AAII 等指标，但这些回答的是"市场情绪怎么样"，而不是"我的仓位可能会亏多少"。

```
当前 L3 温度因子：
  VIX 17.21 → "偏低" → 3 分
  Put/Call 0.47 → "看涨" → 3 分
  AAII +5.36% → "中性" → 3 分
  → 综合 3.42，中性偏乐观
```

**问题：** VIX 17 说明市场不恐慌，但你的 4 只 ETF 在极端情况下最多可能亏多少？L3 回答不了。

### 2. VaR 和 CVaR 是什么？

**用开车类比：**

| 概念 | 开车类比 | 投资含义 |
|---|---|---|
| **波动率** | 这条路平均颠簸程度 | 每天涨跌的幅度 |
| **VaR (风险价值)** | "这条路 95% 的情况下，颠簸不会超过这个程度" | "95% 概率下，一天最多亏 X%" |
| **CVaR (条件风险价值)** | "如果颠簸超过了那个程度，平均会有多颠？" | "如果真的跌破 VaR，平均会亏多少？" |

**具体到你的 ETF：**

假设 513310 中韩半导体的日波动率是 3.5%：

```
VaR(95%, 1天) ≈ 3.5% × 1.65 = 5.8%
→ "95% 的情况下，一天最多亏 5.8%"

CVaR(95%, 1天) ≈ 3.5% × 2.06 = 7.2%
→ "如果真的跌破 5.8%，平均会亏 7.2%"
```

**历史回测验证：** 如果 513310 最近涨了 55%（从 MA120 的 3.30 到 5.12），日波动率可能被近期大涨压低，此时用滚动 60 日 VaR 比静态估计更准确。

### 3. 如何把 VaR/CVaR 放入 L3 评分卡

**新增"尾部风险"子因子：**

| 指标 | 当前值（示例） | 评分标准 | 得分 |
|---|---|---|---|
| 组合 VaR(95%, 1天) | 约 -2.8% | <1%=5, 1-2%=4, 2-3%=3, 3-4%=2, >4%=1 | **3** |
| 组合 CVaR(95%, 1天) | 约 -4.1% | <1.5%=5, 1.5-3%=4, 3-4.5%=3, 4.5-6%=2, >6%=1 | **2** |
| VaR 趋势(20日 vs 60日) | VaR 扩大中 | 缩小=5, 持平=4, 缓慢扩大=3, 加速扩大=2, 爆发=1 | **2** |
| 最大回撤(60日) | ？ | <5%=5, 5-10%=4, 10-15%=3, 15-20%=2, >20%=1 | 待计算 |
| 回撤恢复时间 | ？ | <5天=5, 5-10=4, 10-20=3, 20-40=2, >40=1 | 待计算 |

### 4. 升级后的效果

| 维度 | 升级前 | 升级后 |
|---|---|---|
| 风险认知 | "感觉有点贵" | "组合在 95% 概率下一天最多亏 2.8%" |
| 止损设定 | "MA20 破了就走" | "MA20 距离 = 1.2 倍 CVaR，止损距离合理吗？" |
| 仓位大小 | 固定 55-60% | "组合 VaR 扩大时自动提示降仓" |
| 不同 ETF 比较 | RSI 93 vs 84 | "中韩半导体 VaR 是恒生科技的 3.2 倍" |

### 5. 实现路径

```
Step 1: 用 TickFlow 获取 4 只 ETF 的日线（至少 120 天）
Step 2: 计算每日对数收益率 r = ln(P_t / P_{t-1})
Step 3: VaR(95%) = 收益率分布的第 5 百分位数
Step 4: CVaR(95%) = 低于 VaR 的所有收益率的均值
Step 5: 用滑动窗口（60 日），每天滚一遍，得到 VaR 时间序列
Step 6: 观察 VaR 趋势（扩大/缩小/稳定）
Step 7: 映射到 1-5 评分
```

**预计工作量：** 扩展现有的 `tech_anchor.py` 约 60 行代码。
