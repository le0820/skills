## 二、与三大论点的交叉验证

### 论点1：NVIDIA单卡性能触碰物理极限

**CoreWeave证据：中等支持**

✅ **支持**：
- 电话会中提到"higher component pricing"（元器件成本上升）是CapEx指引上调的原因之一
  - 这意味着：新一代GPU（Rubin）的单卡成本在上升，而不是下降
  - 如果性能翻倍但成本也在翻倍，客户的TCO没有改善
  - NVIDIA正在把物理极限的代价（更大的芯片面积、更复杂的封装、更低的良率）转嫁给客户

✅ **支持**：
- NVIDIA向CoreWeave投资$2B（每股$87.20）→ 芯片供应商需要给自己的客户注资来维持需求
  - 这很像2008年金融危机时银行间的互相救助——供应商在给买家输血
  - 如果NVIDIA芯片需求真的供不应求，为什么需要投资自己的客户？

❌ **不直接支持**：
- CoreWeave本身是纯NVIDIA GPU云，不涉及定制芯片
- 没有直接评论NVIDIA芯片代际性能提升

---

### 论点2：Big 4 Capex是"一锤子买卖"而非永续军备竞赛

**CoreWeave证据：混合信号，但CapEx对收入比率是强烈警告**

🔴 **最震撼的数据：CapEx/收入比率**

```
CoreWeave Q1 2026: CapEx $6.8B / 收入 $2.1B = 3.2×
CoreWeave FY2026: CapEx $31-35B / 收入 $12-13B = 2.5-2.7×
```

这意味着每赚1美元，CoreWeave要花2.5-3.2美元。这放在任何行业都是不可持续的资本结构。

🟡 **收入积压 $99.4B —— 表面看是长期需求的确证**
- 但注意：25-36%在24个月内确认 → 至少$25-36B/2年 ≈ $12.5-18B/年
- 75%在24个月之后 → 意味着大部分合同是2028年及以后的远期承诺
- 这种"远期积压"的结构本身就暗示——**客户在锁定未来的供应，而不是担忧明天的芯片不够用**

🟢 **支持"一锤子买卖"的关键推理**：
- $99.4B积压 + 5年平均合同期 → 客户是"一次签够"，而不是"每年追加"
- 如果客户预期每年都有更强大的芯片问世，为什么要签5年长约锁定今天的架构？
- Meta的$21B承诺、Jane Street的$6B合同、Anthropic的多年代协议——这些都是"一次性大规模采购"的典型特征

⚠️ **不支持"一锤子买卖"的信号**：
- 收入同比增长112% → 说明去年签约的合同正在转化为收入，这不是一次性行为
- 管理层指引2027年末年化收入>$30B → 他们在赌持续增长
- 但注意：$30B年化收入的75%已经签约 → 增长来自已签合同的执行，不是新合同的边际增速

### 论点3：铲子从硬件厂商向模型厂商转移

**CoreWeave证据：强烈支持** 🔥

✅ **CoreWeave的商业逻辑本身就证明了铲子转移**：
- CoreWeave不生产芯片，它只是芯片的"二房东"
- 它的商业模式是：从NVIDIA买GPU → 建数据中心 → 租给AI公司
- **为什么AI公司不直接从NVIDIA买？** 因为NVIDIA GPU太贵、太稀缺、需要专业运维
- CoreWeave的存在本身就是"硬件价值向下游转移"的标志——中间商在攫取NVIDIA和终端用户之间的利润

✅ **NVIDIA $2B入股CoreWeave的深层含义**：
- 这不是NVIDIA第一次投资CoreWeave——而是在持续注资
- $2B = $87.20/股 → NVIDIA在用自己的股票现金买自己客户的股权
- **翻译**：NVIDIA需要确保有人买它的芯片，即使这个"人"的商业模式（花$3.2换$1收入）本身就不可持续
- 如果铲子还在NVIDIA手里，为什么需要注资下游？

✅ **Meta的$21B通过CoreWeave（而非直接向NVIDIA采购）**：
- Meta同时有自研MTIA芯片和CoreWeave/NVIDIA GPU
- 这是"两条腿走路"策略：短期用NVIDIA GPU保证算力，长期用自研ASIC降低依赖
- **一旦MTIA成熟，Meta对CoreWeave/NVIDIA的需求将断崖式下降**

✅ **客户结构变化**：
- Anthropic：同时使用Google TPU + AWS Trainium + CoreWeave（NVIDIA GPU）
- Jane Street（量化交易公司）新签$6B：非AI原生客户进入 → 说明应用层在扩张
- Cohere、Mistral：AI模型公司通过CoreWeave获取算力

## 三、最核心的交叉验证信号

### 🔴 信号1：CapEx黑洞

```
CoreWeave FY2026 CapEx: $31-35B
CoreWeave FY2026 Revenue: $12-13B
差值: ~$20B 需要外部融资

CoreWeave现金: $2.24-3.3B
季度CapEx: $6.8B
现金只能撑 ~1.5个月
```

CoreWeave需要持续的外部融资来维持运营。$8.5B的延迟提取贷款 + NVIDIA的$2B股权投资 + $10B高收益债——这是典型的"烧钱换规模"模式。当音乐停下时，没有椅子的人会很惨。

### 🔴 信号2：JPMorgan一语道破——"投资周期比预期更显著"

分析师原话："investment cycle is 'more pronounced than expected'"

**翻译：我们没意识到这玩意要烧这么多钱。**

这是卖方分析师在委婉地承认：他们的模型低估了AI基础设施投资的资本密集度。而如果连卖方分析师都开始担忧，说明市场的乐观共识正在出现裂缝。

### 🟡 信号3：调整后营业利润率仅1%，全年指引仅~8%

一个"AI云基础设施"公司的调整后营业利润率只有1%（Q1），全年指引仅8%。对比：
- AWS: ~35-38%
- Azure: ~45%+
- Google Cloud: ~30%+

CoreWeave的利润率低得惊人。原因很简单：**GPU采购成本 + 数据中心建设成本 + 电力成本正在吞噬所有毛利。**

这反过来验证了NVIDIA的定价权——$6.8B CapEx中，NVIDIA GPU是最大单项。NVIDIA正在提取整个价值链的绝大部分利润。

### 🔴 信号4：集体诉讼

Alpha Spread报道：律所已发出集体诉讼通知，涉及"可能违反证券法"——针对的是CoreWeave作为上市公司的披露问题。

这虽然不直接影响我们的论点，但说明：当商业模式已经承受巨大压力时，法律风险会被放大。

## 四、综合判断

### 交叉验证矩阵

| 我们的论点 | CoreWeave验证 | 置信度变化 | 关键证据 |
|:-----|:-----:|:-----:|:-----|
| NVIDIA单卡触顶 | 🟡 中等支持 | 维持6/10 | "higher component pricing" + NVIDIA注资下游 |
| 一锤子买卖 | 🟡 混合信号 | 维持5→5.5/10 | $99.4B远期积压结构 + 5年合同期，但收入仍在增长 |
| 铲子转移 | 🟢 强烈支持 | 上调至7/10 | NVIDIA注资客户 + CoreWeave低利润率 + 定制芯片双轨战略 |
| AI Capex不可持续 | 🟢 强烈支持 | 新增 7/10 | CapEx/收入=3.2× + 净亏损率36% + 现金仅够1.5月 |

### 新发现的论点：AI价值链的利润分配极度扭曲

CoreWeave Q1数据揭示了一个残酷的现实：

```
AI价值链利润分配（2026 Q1估算）:

NVIDIA:     毛利率 70-75%  ← 拿走绝大部分利润
CoreWeave:  营业利润率 1%   ← 烧钱维持规模
AI公司:     净亏损          ← 靠融资买算力
终端用户:   免费或低价使用   ← 尚未大规模付费

这条链上只有NVIDIA在赚钱。
```

这个利润分配结构不可持续。最终要么：
- AI公司开始盈利（应用层变现）→ 铲子转移到模型层 ✅
- GPU价格下降（竞争/定制芯片）→ NVIDIA利润被压缩
- CoreWeave等中间商被整合或倒闭 → 行业出清

**所有路径都指向同一个方向：NVIDIA的利润份额必须下降。**
