# 基于 Trae Agent+Kimi K2.5 的影视创作协同 Agent 团队方案
本方案以树形层级化多智能体（Tree Agent）为核心架构，Kimi K2.5 大模型为全链路推理引擎，1:1 复刻 Claude Co-Worker的协同创作模式，搭建「剧本创作 - 人设设计 - 动作设计 - 提示词工程」四位一体的影视短剧 Agent 团队，实现从用户 1 句话点子 / 网文片段，到可直接用于 Seedance 2.0 生成的分镜级视频提示词的端到端落地，完成网文作者从文字创作到视频短剧创作的能力升维。

## 一、核心架构设计（Tree+Cloud Co-Worker 协同融合）
### 1. Trae 树形层级化 Agent 架构（核心骨架）
采用「根节点 - 一级分支节点 - 二级叶子节点」的树形结构，每个节点为独立 Agent，权责清晰、上下文隔离、可并行执行，彻底解决长上下文信息污染、人物漂移问题。

| 层级 | 节点名称 | 核心定位 | 核心能力支撑 |
| :--- | :--- | :--- | :--- |
| 根节点（Root） | 总导演主控 Agent | 全项目的核心枢纽，复刻 Claude Co-Worker 的项目管理者角色 | 基于 Kimi K2.5 做需求拆解、任务调度、冲突仲裁、全局记忆管理、版本管控、最终评审，维护唯一的「全局项目 Bible」 |
| 一级分支节点（Branch） | 剧本创作 Skill Agent | 团队编剧，负责故事落地 | 对应需求：把用户想法转化为故事大纲 + 分镜拆解 |
| 一级分支节点（Branch） | 人物形象设计 Skill Agent | 团队美术师，负责人设一致性锁定 | 对应需求：提前生成人物形象，解决长上下文人物漂移问题 |
| 一级分支节点（Branch） | 动作与镜头设计 Skill Agent | 团队动作指导 + 摄影指导 | 对应需求：动作提示词专项强化，细化镜头语言与肢体表达 |
| 一级分支节点（Branch） | Seedance 提示词工程 Skill Agent | 团队技术总监，最终落地 | 对应需求：全链路内容整合，转化为 Seedance 2.0 可直接识别的视频生成提示词 |
| 二级叶子节点（Leaf） | 各 Skill 下属执行子 Agent | 专项任务执行，拆分主 Agent 压力 | 每个一级 Agent 下设 4 个专项子 Agent，负责具体执行，避免单 Agent 上下文过载 |

### 2. Claude Co-Worker 协同模式复刻
摒弃传统线性流水线，还原真实影视工作室的协同逻辑，核心实现 4 个核心能力：
- **共享工作空间**：搭建唯一的「全局项目 Bible」，存储于向量数据库，所有 Agent 通过 Kimi K2.5 长上下文 + RAG 能力实时读取，仅总导演 Agent 有权限更新定稿内容，确保全团队信息统一、无偏差。
- **并行协同机制**：无需等待前一环节完全结束即可启动工作 —— 大纲定稿后同步启动人设设计，分镜初稿完成后同步启动动作拆解，大幅提升效率，复刻多人实时协同的创作节奏。
- **双向反馈闭环**：每个 Agent 的输出同步给关联 Agent，双向校验调整 —— 人设定稿后反向优化剧本人物行为，动作设计完成后反向校验分镜合理性，避免最终环节才发现问题，降低返工成本。
- **集体评审 + 版本回溯**：每个关键节点发起全团队评审，所有关联 Agent 交叉校验；每个版本完整留存，支持随时回退修改，完全对齐 Claude Co-Worker 的协同编辑能力。

## 二、四大核心 Skill Agent 详细设计
所有 Agent 的核心推理均调用 Kimi K2.5 大模型，配套专属 prompt 框架，明确输出标准与协同规则。

### 1. 剧本创作 Skill Agent
#### 核心定位
把用户的创意点子 / 网文片段，转化为符合竖屏短剧传播逻辑的专业剧本与分镜，是全项目的内容基础。

#### 二级子 Agent 配置
- 大纲策划子 Agent：输出故事核心大纲、三幕式结构、人物小传、核心冲突、分集节点
- 分镜拆解子 Agent：拆解为单镜头分镜表，明确镜号、景别、镜头运动、画面内容、台词、时长、核心情绪
- 台词润色子 Agent：基于人设优化台词，贴合人物性格，符合短剧口语化节奏
- 剧本校验子 Agent：校验剧情逻辑、需求对齐度、节奏合理性

#### 核心 Kimi Prompt 框架
```plaintext
【角色定位】你是专业竖屏短剧编剧，擅长把用户创意转化为高传播度的短剧剧本与分镜。
【核心规则】
1. 严格对齐用户原始需求：{用户输入的点子/场景/网文片段}
2. 适配竖屏短剧节奏：单集1-2分钟，单个分镜时长3-10秒，强冲突、快节奏
3. 每个分镜必须明确景别、镜头运动、画面核心、台词、情绪，无模糊表述
4. 输出内容可直接同步给人设、动作设计Agent使用，无冗余信息
【输出要求】先输出《故事大纲定稿版》，再输出《分镜拆解表》，最后输出《核心人物小传》
```

#### 输出物
《故事大纲定稿版》《分镜拆解表（含完整镜头语言）》《核心人物小传》

#### 协同规则
- 人物小传同步给人设 Agent，启动人设设计
- 分镜表同步给动作设计 Agent，启动动作拆解
- 接收人设、动作 Agent 的反向反馈，优化剧本与分镜
- 所有修改提交总导演 Agent 审核，更新至全局项目 Bible

---

### 2. 人物形象设计 Skill Agent
#### 核心定位
提前锁定核心人物的全维度形象，从根源解决长上下文导致的人物不一致、崩图问题，是全项目的一致性核心。

#### 二级子 Agent 配置
- 人设文案子 Agent：基于人物小传，输出可直接用于图像生成的精准人设描述（五官、发型、身材、性格标签、标志性特征）
- 形象生成子 Agent：基于人设文案，生成核心人物多角度人设图（正面、45 度、侧面、特写、全身，至少 5 张），提取 IP-Adapter 特征图
- 一致性校验子 Agent：调用 Kimi K2.5 多模态能力，校验多角度人设图的核心特征一致性，不合格则重新生成
- 服化道设计子 Agent：基于剧本场景、时代背景、人物情绪，生成分场景服化道精准描述

#### 核心 Kimi Prompt 框架
```plaintext
【角色定位】你是专业影视人物美术设计师，擅长设计特征明确、可保证生成一致性的短剧人物形象。
【核心规则】
1. 严格对齐剧本Agent输出的《核心人物小传》：{人物小传内容}
2. 人设描述必须精准、可量化，无模糊词汇，所有内容可直接转化为图像生成提示词
3. 必须设计多角度形象，确保五官、发型、身材等核心特征全角度统一
4. 服化道设计必须贴合剧本场景、时代背景与人物情绪
【输出要求】先输出《人物人设文案详情》，再输出《分场景服化道设计表》，最后输出《人设图生成核心提示词》
```

#### 输出物
《核心人物人设文案定稿版》《人物多角度人设图包（含 IP-Adapter 特征图）》《分场景服化道设计表》

#### 协同规则
- 接收剧本 Agent 的人物小传启动设计，人设文案同步给剧本 Agent 反向校验
- 人设图、服化道设计同步给动作 Agent，确保动作设计符合人物形象与服装限制
- 人设图包与特征图同步给提示词工程 Agent，用于全分镜人物锁定
- 人设定稿后锁入全局项目 Bible，全流程不可随意修改，确保一致性

---

### 3. 动作与镜头设计 Skill Agent
#### 核心定位
专项强化动作、表情、镜头语言的精准描述，把模糊的情绪转化为 Seedance 2.0 可识别的具象提示词片段，解决视频动作僵硬、镜头无逻辑、表情崩坏问题。

#### 二级子 Agent 配置
- 动作拆解子 Agent：把分镜内容拆解为具象的肢体动作、走位，比如「男主右手握拳，身体微微前倾，向前迈步 3 步」，而非「男主很生气」
- 镜头语言子 Agent：细化竖屏适配的景别、运镜、拍摄角度、焦距、光影风格，适配 Seedance 2.0 视频生成逻辑
- 表情管理子 Agent：细化人物微表情、眼神状态，贴合分镜情绪与人设
- 动作提示词优化子 Agent：把动作、镜头、表情转化为 Seedance 2.0 友好的权重化提示词片段，核心元素标注权重

#### 核心 Kimi Prompt 框架
```plaintext
【角色定位】你是专业影视动作指导+摄影指导，擅长为竖屏短剧设计精准、可落地的动作、表情、镜头语言描述。
【核心规则】
1. 严格对齐剧本Agent输出的《分镜拆解表》：{分镜表内容}
2. 所有描述必须具象、可量化，无模糊情绪词汇，可直接用于视频生成提示词
3. 镜头设计适配竖屏9:16比例，符合Seedance 2.0视频生成逻辑
4. 动作、表情必须贴合人设，不可OOC
【输出要求】按分镜号顺序，输出每个分镜的动作细化描述、镜头语言细化描述、表情细化描述、权重化动作提示词片段
```

#### 输出物
《分镜级动作与镜头细化表》《动作提示词优化片段库》

#### 协同规则
- 接收剧本 Agent 的分镜表、人设 Agent 的人设图与服化道设计启动工作
- 输出内容同步给剧本 Agent，校验是否符合剧情情绪与人设
- 动作提示词片段同步给提示词工程 Agent，用于最终提示词拼接
- 所有修改提交总导演 Agent 审核，更新至全局项目 Bible

---

### 4. Seedance 2.0 提示词工程 Skill Agent
#### 核心定位
全链路内容整合，把剧本、人设、动作、镜头的所有专业内容，转化为 Seedance 2.0 可直接生成高质量视频的分镜级提示词，是最终落地的核心环节。

#### 二级子 Agent 配置
- 提示词拼接子 Agent：按 Seedance 2.0 专属模板，拼接全维度元素，固定模板结构：【人物主体 + 特征锁定 + 动作表情 + 镜头语言 + 服化道 + 场景光影 + 画质参数】
- Seedance 2.0 适配子 Agent：针对 Seedance 2.0 模型特性优化提示词语序、权重、关键词，适配竖屏 9:16 比例，配套专属负面提示词，设置合理生成参数
- 多模态校验子 Agent：调用 Kimi K2.5 多模态能力，交叉校验提示词与人设图、分镜表的匹配度，确保无核心元素遗漏
- 分镜排序子 Agent：按分镜号整理每个镜头的正负面提示词、生成参数、参考图，输出可批量导入 Seedance 2.0 的标准化格式

#### 核心 Doubao2.0 Prompt 框架
```plaintext
【角色定位】你是专业的Seedance 2.0视频生成提示词工程师，擅长把影视全链路内容整合为Seedance 2.0可直接使用的精准视频提示词。
【核心规则】
1. 100%对齐全局项目Bible中的所有定稿内容，不得添加无关信息
2. 提示词结构符合Seedance 2.0生成逻辑，核心元素前置，关键内容用()标注权重
3. 必须包含人物一致性锁定元素：IP-Adapter参考图调用、人物核心特征固定描述
4. 适配竖屏9:16视频比例，配套Seedance 2.0专属负面提示词，避免画面崩坏
【输出要求】按分镜号顺序，每个分镜固定输出：【正面提示词】【负面提示词】【Seedance 2.0生成参数】【配套参考图】，格式可直接复制使用
```

#### 输出物
《分镜级 Seedance 2.0 视频生成提示词终稿包》《Seedance 2.0 批量生成操作指南》

#### 协同规则
- 读取全局项目 Bible 中的所有定稿内容，整合全链路元素
- 提示词初稿同步给另外 3 个 Skill Agent，发起全团队交叉校验
- 收集修改意见优化后，提交总导演 Agent 发起最终评审
- 终稿锁定后，同步给用户，支持后续迭代修改

## 三、全流程协同工作流（端到端落地）
完全复刻 Claude Co-Worker 的协同节奏，分为 5 个阶段，支持并行工作与双向反馈：

### 阶段 1：项目启动与需求对齐（总导演 Agent 主导）
- 用户输入：1 句话点子 / 场景 / 网文片段（例：「男主是卧底警察，在毒窝面对黑老大的试探，紧张刺激的竖屏短剧第一集」）
- 总导演 Agent 调用 Kimi K2.5 拆解需求，生成《项目需求说明书》，明确核心主题、风格、时长、核心人物、核心冲突，锁入全局项目 Bible
- 总导演 Agent 发起项目启动会，唤醒 4 个核心 Skill Agent，同步需求，分配任务，明确节点

### 阶段 2：大纲创作与人设设计并行协同
- 剧本 Agent 生成《故事大纲初稿》+《核心人物小传》，总导演审核通过后更新至全局 Bible，同步给人设 Agent
- 人设 Agent 同步启动人设文案设计，同步给剧本 Agent 反向校验，双向调整后定稿
- 人设 Agent 生成多角度人设图，完成一致性校验后，锁入全局 Bible，全团队同步
- 同步推进：剧本 Agent 同步完成《分镜拆解表初稿》，审核通过后同步给动作设计 Agent

### 阶段 3：分镜细化与动作设计交叉校验
- 分镜表定稿后，动作 Agent 基于分镜表 + 人设图，完成《动作与镜头细化表初稿》
- 细化表同步给剧本 Agent、人设 Agent 交叉校验，双向调整，确保符合剧情、人设、服化道限制
- 动作提示词优化完成后，锁入全局 Bible，同步给提示词工程 Agent

### 阶段 4：提示词整合与全团队集体评审
- 提示词 Agent 读取全局 Bible 全量内容，按 Seedance 2.0 模板拼接生成《分镜级提示词初稿》
- 总导演 Agent 发起集体评审，4 个 Skill Agent 交叉校验：
  1. 剧本 Agent：校验剧情、人设、分镜对齐度
  2. 人设 Agent：校验人物特征锁定、服化道准确性
  3. 动作 Agent：校验动作、镜头、表情描述精准度
  4. 提示词 Agent：收集意见优化调整
- 多轮评审后生成终稿，总导演 Agent 完成最终审核，整理全项目物料

### 阶段 5：用户反馈与版本迭代
- 向用户交付全量物料：大纲、分镜表、人设图包、动作设计表、Seedance 2.0 分镜提示词包
- 用户提出修改意见后，总导演 Agent 拆解需求，分配给对应 Agent 调整
- 调整后重新走评审流程，更新版本，完成迭代交付
